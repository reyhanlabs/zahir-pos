Pendahuluan
Introduction
Apa itu POS ERP dan apa saja fiturnya?
What is POS ERP and what are its features?

POS ERP adalah aplikasi Point of Sales berbasis web yang berjalan sepenuhnya di browser. Data disimpan secara real-time di Firebase Firestore milik Anda sendiri sehingga data toko 100% berada di bawah kendali Anda.

POS ERP is a web-based Point of Sales application running entirely in the browser. Data is stored in real-time on your own Firebase Firestore, so your store data is 100% under your control.

Setup Awal (Pertama Kali)
Initial Setup (First Time)
Konfigurasi Firebase dan pembuatan database toko
Firebase configuration and store database creation
ℹ️
Setup hanya dilakukan sekali Setup is done only once Setelah setup selesai, konfigurasi tersimpan di browser. Buka ulang aplikasi langsung ke halaman login. After setup is complete, the config is saved in the browser. Reopening the app goes directly to the login page.
1
Buka Aplikasi & Isi Konfigurasi Firebase
Open App & Fill Firebase Configuration
Saat pertama buka aplikasi, Anda akan melihat form Konfigurasi Firebase. Isi 6 field: API Key, Auth Domain, Project ID, Storage Bucket, Messaging Sender ID, dan App ID. Semua nilai ini didapat dari Firebase Console.
When first opening the app, you'll see the Firebase Configuration form. Fill in 6 fields: API Key, Auth Domain, Project ID, Storage Bucket, Messaging Sender ID, and App ID. All values come from the Firebase Console.
2
Isi Informasi Toko
Fill Store Information
Masukkan nama toko/bisnis, nama cabang/outlet, mata uang (Rupiah, Dollar, dll), dan catatan yang akan muncul di bagian bawah struk.
Enter your store/business name, branch/outlet name, currency (Rupiah, Dollar, etc.), and a note that will appear at the bottom of receipts.
3
Buat Akun Admin Pertama
Create First Admin Account
Masukkan nama lengkap, email, dan password (min. 6 karakter). Akun ini akan menjadi Superadmin dengan akses penuh ke semua fitur termasuk manajemen user.
Enter full name, email, and password (min. 6 characters). This account becomes the Superadmin with full access to all features including user management.
4
Klik "Buat Database & Mulai"
Click "Create Database & Start"
Aplikasi akan membuat akun di Firebase Auth, menyimpan konfigurasi toko di Firestore, dan langsung login otomatis. Proses berlangsung beberapa detik.
The app creates the account in Firebase Auth, saves the store config in Firestore, and logs in automatically. The process takes a few seconds.
⚠️
Sebelum setup, pastikan di Firebase Console: Before setup, ensure in Firebase Console: 1) Authentication → Sign-in method → Email/Password → Aktifkan  2) Firestore Database → Buat database → Mode Production → Atur rules 1) Authentication → Sign-in method → Email/Password → Enable  2) Firestore Database → Create database → Production Mode → Set rules

Point of Sales — Kasir
Point of Sales — Cashier
Cara melakukan transaksi penjualan
How to process a sales transaction
1
Cari / Pilih Produk
Search / Select Product
Gunakan kolom pencarian atau klik chip kategori untuk memfilter produk. Klik kartu produk untuk menambahkan ke keranjang. Stok produk habis tidak akan muncul.
Use the search bar or click category chips to filter products. Click a product card to add it to the cart. Out-of-stock products won't appear.
2
Atur Keranjang
Adjust Cart
Di panel kanan, gunakan tombol + dan − untuk mengubah jumlah item. Klik tombol × untuk menghapus item. Klik "Kosongkan" untuk mengosongkan seluruh keranjang.
In the right panel, use + and − buttons to change quantity. Click × to remove an item. Click "Clear" to empty the entire cart.
3
Isi Diskon & Pilih Customer (Opsional)
Set Discount & Select Customer (Optional)
Masukkan persentase diskon (0–100%) di kolom Diskon (%). Pilih nama customer dari dropdown jika transaksi ingin dikaitkan ke customer tertentu.
Enter a discount percentage (0–100%) in the Discount (%) field. Select a customer name from the dropdown to link the transaction to a specific customer.
4
Klik "Bayar Sekarang"
Click "Pay Now"
Modal pembayaran akan terbuka menampilkan total yang harus dibayar.
The payment modal opens showing the total amount due.
5
Pilih Metode Pembayaran
Select Payment Method
Tersedia 4 metode: Tunai, Transfer Bank, QRIS, dan Kartu Debit/Kredit. Untuk Tunai, masukkan uang yang diterima — kembalian dihitung otomatis.
4 methods available: Cash, Bank Transfer, QRIS, and Debit/Credit Card. For Cash, enter the amount received — change is auto-calculated.
6
Konfirmasi & Cetak Struk
Confirm & Print Receipt
Klik "Konfirmasi Bayar". Stok produk otomatis dikurangi, transaksi tersimpan di Firestore, dan struk digital tampil. Klik "Cetak Struk" untuk mencetak via printer.
Click "Confirm Payment". Product stock auto-decrements, the transaction is saved to Firestore, and a digital receipt appears. Click "Print Receipt" to print via printer.
⚠️
Catatan Stok Stock Note Stok yang ditampilkan di POS adalah stok saat halaman dimuat. Jika ada transaksi paralel, refresh halaman untuk stok terkini. Stock shown in POS is as of page load. If parallel transactions occur, refresh the page for updated stock.

FAQ
Pertanyaan yang sering ditanyakan
Frequently asked questions
?
Apakah data saya aman?
Is my data safe?
Ya. Data disimpan di Firebase Firestore milik Anda sendiri. Anthropic atau pihak ketiga tidak memiliki akses ke database Anda.
Yes. Data is stored in your own Firebase Firestore. Anthropic or any third party has no access to your database.
?
Bisakah digunakan di beberapa perangkat?
Can it be used on multiple devices?
Ya. Karena berbasis web dan Firebase, Anda bisa login dari PC, tablet, atau HP selama ada browser dan koneksi internet.
Yes. Since it's web-based and Firebase-powered, you can log in from PC, tablet, or phone as long as you have a browser and internet connection.
?
Bagaimana jika saya lupa password?
What if I forget my password?
Reset password melalui Firebase Console → Authentication → Users → Reset password untuk mengirim email reset ke pengguna.
Reset the password via Firebase Console → Authentication → Users → Reset password to send a reset email to the user.
?
Bisakah aplikasi digunakan offline?
Can the app be used offline?
Saat ini belum. Koneksi internet diperlukan untuk sinkronisasi data dengan Firebase Firestore secara real-time.
Not yet. An internet connection is required for real-time data sync with Firebase Firestore.
?
Bagaimana cara backup data?
How to back up data?
Gunakan fitur export di Firebase Console → Firestore → Import/Export. Data dapat diekspor ke Cloud Storage secara berkala.
Use the export feature at Firebase Console → Firestore → Import/Export. Data can be exported to Cloud Storage periodically.
?
Apakah ada biaya untuk Firebase?
Is there a cost for Firebase?
Firebase menawarkan Spark Plan (gratis) yang cukup untuk toko kecil-menengah: 20K read/hari, 20K write/hari. Upgrade ke Blaze Plan jika volume transaksi sangat tinggi.
Firebase offers a free Spark Plan sufficient for small-medium stores: 20K reads/day, 20K writes/day. Upgrade to Blaze Plan for very high transaction volumes.

Panduan Setup Firebase
Firebase Setup Guide
Cara mendapatkan konfigurasi Firebase untuk aplikasi ini
How to get Firebase configuration for this app
1
Buat Project Firebase
Create Firebase Project
Buka console.firebase.google.com → klik "Add Project" → ikuti wizard → aktifkan/nonaktifkan Google Analytics sesuai kebutuhan.
Go to console.firebase.google.com → click "Add Project" → follow the wizard → enable/disable Google Analytics as needed.
2
Aktifkan Authentication Email/Password
Enable Email/Password Authentication
Di sidebar Firebase → Authentication → Get started → Sign-in method → Email/Password → Enable → Save.
In Firebase sidebar → Authentication → Get started → Sign-in method → Email/Password → Enable → Save.
3
Buat Firestore Database
Create Firestore Database
Di sidebar → Firestore Database → Create database → Start in production mode → Pilih lokasi server → Create. Pilih asia-southeast1 (Singapura) untuk Indonesia.
In sidebar → Firestore Database → Create database → Start in production mode → Choose server location → Create. Choose asia-southeast1 (Singapore) for Southeast Asia.
4
Atur Firestore Rules
Set Firestore Rules
Di Firestore → tab Rules → ganti isi dengan rules berikut lalu klik Publish:
In Firestore → Rules tab → replace content with the following rules, then click Publish:
// Firestore Security Rules — POS ERP rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
5
Daftarkan Web App & Ambil Config
Register Web App & Get Config
Di sidebar → Project Settings (⚙) → Your Apps → Web (</>) → Register App → beri nama app → klik Register. Setelah itu salin nilai firebaseConfig yang tampil.
In sidebar → Project Settings (⚙) → Your Apps → Web (</>) → Register App → give the app a name → click Register. Then copy the displayed firebaseConfig values.
// Contoh firebaseConfig yang perlu disalin // Example firebaseConfig to copy const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "my-project.firebaseapp.com", projectId: "my-project-id", storageBucket: "my-project.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" };

Deploy ke Vercel via GitHub
Deploy to Vercel via GitHub
Cara meng-hosting aplikasi agar bisa diakses online
How to host the app for online access
1
Upload ke GitHub
Upload to GitHub
Buka github.com → buat repository baru (bisa Public atau Private) → upload file pos-app.html ke repository. Anda bisa rename file menjadi index.html.
Go to github.com → create a new repository (Public or Private) → upload pos-app.html to the repository. You may rename the file to index.html.
2
Connect ke Vercel
Connect to Vercel
Buka vercel.com → login/daftar → klik "New Project" → pilih "Import Git Repository" → pilih repo yang baru dibuat.
Go to vercel.com → login/register → click "New Project" → select "Import Git Repository" → choose the newly created repo.
3
Deploy
Deploy
Di halaman konfigurasi Vercel, langsung klik "Deploy" tanpa perlu mengubah apapun. Vercel otomatis mendeteksi file HTML statis. Proses deploy berlangsung 30–60 detik.
On the Vercel configuration page, just click "Deploy" without changing anything. Vercel auto-detects the static HTML file. Deployment takes 30–60 seconds.
4
Akses URL Publik
Access Public URL
Setelah deploy selesai, Vercel memberikan URL seperti your-app.vercel.app. Buka URL tersebut → lakukan setup Firebase pertama kali → selesai!
After deployment, Vercel provides a URL like your-app.vercel.app. Open that URL → do the first-time Firebase setup → done!
5
Auto-Deploy Selanjutnya
Future Auto-Deploys
Setiap kali Anda push perubahan ke GitHub, Vercel otomatis re-deploy aplikasi. Tidak perlu langkah manual lagi.
Every time you push changes to GitHub, Vercel automatically re-deploys the app. No manual steps needed.
💡
Domain Kustom (Opsional) Custom Domain (Optional) Di Vercel → Settings → Domains → tambahkan domain kustom Anda (contoh: pos.tokobatik.com) untuk tampilan lebih profesional. In Vercel → Settings → Domains → add your custom domain (e.g. pos.mystore.com) for a more professional appearance.
🚨
Tambahkan Domain Vercel ke Firebase Authorized Domains Add Vercel Domain to Firebase Authorized Domains Di Firebase Console → Authentication → Settings → Authorized domains → klik Add domain → masukkan domain Vercel Anda (contoh: your-app.vercel.app). Tanpa ini, login akan gagal. In Firebase Console → Authentication → Settings → Authorized domains → click Add domain → enter your Vercel domain (e.g. your-app.vercel.app). Without this, login will fail.
